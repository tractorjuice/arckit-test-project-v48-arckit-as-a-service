## Summary

AI provider abstraction is the architectural pattern that lets a SaaS swap LLM providers in days rather than weeks (ArcKit Goal G-8: ≤ 5 working-day swap). It is implemented by wrapping every LLM call in a provider-agnostic AIAdaptor that exposes a stable internal interface (prompt → completion + provenance metadata) and is backed by per-provider implementations (Bedrock, Azure OpenAI, Vertex AI, Llama-3 / Mistral self-host). ArcKit's adaptor is mandated by ADR-004 with two providers from day one.

## Key Findings

- **Two providers from day one is non-negotiable**. Single-provider AI integration creates an existential lock-in for any SaaS whose value proposition depends on AI generation (FR-004). ArcKit's ADR-004 mandates a primary (Bedrock) and a CI-validated secondary (Azure OpenAI), with quarterly CI runs on the secondary asserting feature parity.
- **UK-region availability is the gating criterion**. As of 2026-05-03, Bedrock in eu-west-2 London hosts Anthropic Claude 3.5 Sonnet / Claude 3 Haiku and Amazon Nova; Azure OpenAI in UK South hosts GPT-4o / GPT-4 Turbo / GPT-3.5 Turbo; Vertex AI in europe-west2 London hosts Gemini 1.5 Pro / Flash. Other UK regions and other model families lag — re-test at Beta gate.
- **Per-tenant budget caps are the cost-discipline primary control**. AI inference is the largest single OpEx variable (R-006: residual 9, at appetite ceiling). ADR-008 establishes monthly per-tenant inference budgets (calls + tokens) with soft warnings at 80% and hard throttle at 100%. The free SME tier's cap is what makes Principle 1 affordable at GA-scale.
- **Tier-aware model routing reduces unit cost**. Route SME free tier to cheaper models (Claude 3 Haiku, GPT-3.5 Turbo, Amazon Nova); route paid SME and large-enterprise tiers to higher-quality models (Claude 3.5 Sonnet, GPT-4o). Tenants with sensitive content can be routed to UK-only sub-processors only.
- **No-train-by-default is contractually mandatory**. Both Bedrock and Azure OpenAI offer contractual no-train-on-customer-content posture; the AIAdaptor must record provenance metadata (provider, model, model version, prompt, timestamp, tokens) on every generation event so the FR-004 lineage requirement and DPIA evidence are satisfied.
- **Sovereign mode is a different AIAdaptor implementation**. ADR-004 supports `none` (manual authoring only), `local-openweight` (Llama-3 / Mistral on customer-controlled inference), and `customer-approved-endpoint` (e.g., a tenant's own approved Bedrock or Azure OpenAI deployment). Default sovereign profile points at `none` so no external dependency is mandatory.
- **Provenance metadata is non-optional**. Every generation event must record `command`, `model`, `model_version`, `prompt`, `prompt_hash`, `timestamp`, `input_tokens`, `output_tokens`, `tenant_id`. This is the FR-005 / Principle 9 lineage commitment and the basis for AI-Playbook / ATRS evidence.

## Relevance to Projects

- **ArcKit as a Service Project 001 (managed SaaS)** — primary application; Bedrock + Azure OpenAI per ADR-004 / Goal G-8.
- **ArcKit as a Service Project 002 (sovereign deployment)** — same AIAdaptor, different implementations: `none` default, `local-openweight` and `customer-approved-endpoint` available.
