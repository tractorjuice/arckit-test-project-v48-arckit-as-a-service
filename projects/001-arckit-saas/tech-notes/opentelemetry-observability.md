## Summary

OpenTelemetry (OTel) is the open-standard instrumentation API and SDK for application telemetry — logs, metrics, and traces — adopted by ArcKit as a Service Project 001 per ADR-005. OTel separates *instrumentation* (in app code) from *export* (where the telemetry lands). ArcKit instruments services with OTel SDKs once and exports to Grafana Cloud (managed UK/EU) for the SaaS route, or to a customer-controlled collector for the Project 002 sovereign route — without changing app code.

## Key Findings

- **Instrument once, export anywhere**. The OpenTelemetry collector is the architectural seam: app code emits OTel; the collector routes signals to Grafana Cloud / Datadog / customer-controlled backend. ArcKit's SaaS route uses the collector to ship to Grafana Cloud (UK/EU); ArcKit's sovereign route ships to a customer-supplied collector address. No app-code change between the two.
- **`tenant_id` is a first-class resource attribute**. OpenTelemetry's resource-attribute model lets `tenant_id`, `cell_id`, `service`, `environment` be propagated by the SDKs on every span / metric / log emission. ArcKit middleware injects `tenant_id` at request entry; SDKs propagate it through every downstream call. This is the basis for tenant-scoped FR-012 audit access.
- **PII redaction at source is mandatory**. UK GDPR requires logs not to leak personal data. ArcKit applies redaction filters at the OpenTelemetry collector before egress — never relying on backend-side redaction (which is reactive and audit-trailed only after exposure). Field-level redaction is configured per artefact type.
- **Tiered retention matches the regulatory regime**. Application logs: 90 days for troubleshooting (NFR-C-002 floor). Security and audit logs: 12 months minimum (Principle 6); 7 years for SME-verification decisions (BR-003). Metrics: 13 months for year-on-year comparison. The collector tags signals with their retention class; backend lifecycle policies enforce.
- **SLO-driven alerting is the noise floor**. Burn-rate alerts on the NFR-A-001 99.9% SLO; tenant-aware error budgets prevent one tenant's bad behaviour from triggering platform-wide pages. Grafana SLO and Grafana OnCall are the recommended SaaS-route tooling.
- **Audit-log immutability requires storage-layer enforcement**. Tenant audit log (FR-012) and operator audit log (FR-014) lands in S3 Object Lock (Compliance Mode) or Azure Blob immutable storage. Application-layer compromise cannot tamper. Compliance Mode is FINRA / SEC 17a-4(f) verified, the strongest objective evidence for NCSC CAF B5 / C1.
- **Sovereign mode never phones home**. Project 002 deploys with the OTel collector pointing at a customer-controlled address only. No vendor-controlled telemetry endpoint is hard-coded; the runtime emits, the customer collects.

## Relevance to Projects

- **ArcKit as a Service Project 001 (managed SaaS)** — applied via Grafana Cloud (UK/EU) per ADR-005 / Category 3 of `ARC-001-RSCH-v1.0.md`.
- **ArcKit as a Service Project 002 (sovereign deployment)** — same instrumentation, customer-controlled backend; the open-source Grafana / Loki / Tempo / Mimir self-host equivalent.
