# ArcKit as a Service

ArcKit as a Service for UK Government — a managed, multi-tenant offering of the ArcKit enterprise architecture governance toolkit, designed for UK public sector adoption (GDS Service Standard, TCoP, NCSC CAF, G-Cloud), with a parallel sovereign / air-gapped deployment route for MOD and sensitive sites.

This repository is itself an ArcKit-governed project: it contains the principles, requirements, stakeholders, ADRs, designs, risk register, business case, compliance assessments, and operational packs for the service.

## Overview

This repository uses **ArcKit v4.12.3** for enterprise architecture governance and documentation.

It is organised as **two related projects** built from a single codebase per Principle 21:

- **`projects/001-arckit-saas/`** — managed multi-tenant SaaS offering for UK SMEs supplying Government.
- **`projects/002-arckit-sovereign/`** — sovereign / air-gapped deployment route for MOD and sensitive sites.

Cross-project artefacts (principles, Wardley map, glossary, executive strategy) live under `projects/000-global/`.

## Documentation site

Open `docs/index.html` in a browser for the interactive dashboard, sidebar navigation, KPI cards, governance checklist, and Mermaid-rendered diagrams. The site is generated by `/arckit:pages` and reads `manifest.json` + `health.json` at runtime.

## Getting Started

1. Open this repo in a GitHub Codespace (or clone locally)

4. Restart Claude Code once to resolve the marketplace plugin

## How this project was created

Every artefact in this repository was generated end-to-end by **Claude Opus 4.7 (1M context)** using the ArcKit plugin's slash commands, then committed in batched, traceable commits. The build below is the actual sequence that produced what you see in `git log`.

### Phase 1 — Foundations (cross-project)

Established the architectural anchor that every later artefact traces back to:

```bash
/arckit:init                      # repo scaffold + .claude/settings + projects/ tree
/arckit:principles                # global Architecture Principles (v1.0)
/arckit:principles                # v2.0: dual deployment model + sovereign Principle 21
```

Output: `projects/000-global/ARC-000-PRIN-v1.0.md`, `ARC-000-PRIN-v2.0.md`.

## Document naming convention

All documents follow: `ARC-{PROJECT_ID}-{TYPE_CODE}-v{VERSION}.md`

- Example: `ARC-001-REQ-v1.0.md` — Requirements for project 001.
- Multi-instance types: `ARC-001-ADR-001-v1.0.md`, `ARC-002-DIAG-001-v1.0.md`.

Every document has a Document Control header (ID, type, classification, status, owner, version, review cycle), a Revision History, and traceability links to the artefacts it depends on or supersedes.
